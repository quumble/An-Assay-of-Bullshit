# Claude_Analysis — re-analysis of Bullshit Assay 2

This directory contains a re-analysis of the Bullshit Assay 2 data
conducted with Claude Opus 4.7 assistance. The original as-run
analysis artifacts under `../Results and Analysis/` are preserved
unchanged. Outputs in this directory supersede them for Q2 and Q3.

## Why this exists

Three things prompted the re-analysis:

1. The as-run analyzer had a key-name mismatch with the feature
   serializer: it looked up `noun_ratio` etc. while the serializer
   wrote `pos_noun_ratio` etc. The five POS ratio features were
   silently absent from Q2 and Q3 in the original report. The
   preregistration named all five as primary features; this fix
   brings the analyzer into compliance with the preregistration.
2. A floating-point clamp was needed in the cosine distance
   computation to run on current sklearn (≥1.6); pure numerical
   correctness, no effect on results.
3. A sensitivity analysis under strict-yes-only validation scoring
   was added, addressing a deviation from the locked coding
   heuristic that did not specify how partial-credit tags map to a
   numeric agreement rate.

Full audit trail in `deviations.md`.

## Headline result of the correction

The five POS ratio features were entirely missed in the original
report. With them included, `pos_pronoun_ratio` is the largest
effect in the study (η² = 0.229, ~6 points larger than any other
feature). Four other POS features clear Bonferroni significance and
the η² > 0.01 threshold. Effect sizes for the eight previously-
present features are byte-identical to the original report.

## Robustness

Under the strict-yes-only validation scoring sensitivity (which
would gate out the POS ratios and rhyme participation rate, leaving
7 of 13 features), every retained feature still differs across
models with Bonferroni correction and η² > 0.01. Q1 silhouettes are
identical. Q3 phrasing main effect remains significant. The headline
"models differ on every preregistered structural feature" survives
the strictest interpretation.

## Files

- `analysis_report.md` — corrected primary analysis report.
  Generated by `../analysis/analyze.py` after the bug fix.
- `analysis_results.json` — corresponding JSON results.
- `deviations.md` — log of every change made in the re-analysis,
  with rationale and effect on conclusions.
- `sensitivity_strict_validation/` — Q1–Q4 rerun with the 7-feature
  reduced battery (no POS ratios, no rhyme participation rate). Used
  to verify that conclusions hold under the strictest interpretation
  of the validation gate.

## How to reproduce

The fixes are in `../analysis/analyze.py` (or `../analyze.py`, which
is a duplicate). Reproducing the corrected report:

```
cd Bullshit_Assay_2
python run_analysis.py analyze \
  --features "Results and Analysis/features_final_5397.jsonl" \
  --embeddings <path-to-embeddings.jsonl> \
  --report-md Claude_Analysis/analysis_report.md \
  --report-json Claude_Analysis/analysis_results.json
```

Note that `embeddings.jsonl` is too large for GitHub (~330MB) and is
not committed. It is regenerable from `features.jsonl` via
`python run_analysis.py embed`, at an OpenAI API cost of roughly $2
for 5,397 poems against `text-embedding-3-large`.

## Relationship to the as-run record

The directory `../Results and Analysis/` is preserved unchanged as
the as-run record. The corrected outputs here supersede that
directory only for Q2 and Q3 quantitative claims. Q1 silhouettes
and Q4 compliance are byte-identical between the two and either may
be cited.
